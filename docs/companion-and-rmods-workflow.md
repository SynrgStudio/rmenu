# rMenu companions and rMods workflow

This document describes the current native integration model for rMenu, RSnip, RTasks, and rMods.

## Data root

rMenu keeps product-owned mutable data outside the application directory.

Default Windows data root:

```text
C:\rMenuData
```

Layout:

```text
C:\rMenuData\
  modules\
  companions\
    rsnip\
      rsnip.exe
      config\
      state\
      logs\
    rtasks\
      rtasks.exe
      config\
      state\
      logs\
  config\
  state\
    modules\
      <module-id>\
```

Overrides:

- `--data-dir <PATH>` or `RMENU_DATA_DIR` changes the data root.
- `--modules-dir <PATH>` or `RMENU_MODULES_DIR` explicitly overrides module discovery.
- If no explicit module directory is set, modules load from `<data_dir>\modules`.

## Daemon lifecycle

Default daemon hotkey:

```text
Ctrl+Shift+Space
```

Start:

```powershell
rmenu-daemon.exe
```

Stop:

```powershell
rmenu-daemon.exe --quit
```

`--quit` also asks managed companions to shut down. RSnip has a fallback process kill if IPC shutdown does not complete.

## RSnip companion

RSnip owns screenshot, recording, and OCR behavior. rMenu only exposes native command aliases and dispatches over IPC.

Managed install target:

```text
<data_dir>\companions\rsnip\rsnip.exe
```

Discovery order:

1. `<data_dir>\companions\rsnip\rsnip.exe`
2. `RMENU_RSNIP_PATH`
3. `C:\rSnip\target\release\rsnip.exe`
4. unambiguous `PATH` match

Install/update:

```text
/install rsnip
```

or:

```powershell
rmenu.exe --install rsnip
```

Latest asset:

```text
https://github.com/SynrgStudio/rSnip/releases/latest/download/rsnip.exe
```

User-facing aliases:

```text
snip  -> RSnip screenshot region
rec   -> RSnip screen recording
ocr   -> RSnip OCR region
```

The old `.rmod` PowerShell/CMD wrappers are not used.

## RTasks companion

RTasks is used as a backend/panel companion. rMenu owns task input.

Managed install target:

```text
<data_dir>\companions\rtasks\rtasks.exe
```

Install/update:

```text
/install rtasks
```

or:

```powershell
rmenu.exe --install rtasks
```

Latest asset:

```text
https://github.com/SynrgStudio/rtasks/releases/latest/download/rtasks.exe
```

### Embedded task input

Typing `t ` enters RTasks input mode:

```text
t comprar pan mañana
```

Shortcuts while in task mode:

```text
Alt+1  toggle TODO
Alt+2  toggle DOING
Alt+3  toggle DONE
Alt+Q  toggle high priority
Alt+W  toggle medium priority
Alt+E  toggle low priority
```

Pressing the same status/priority shortcut again clears it.

Enter sends `add_task` to the RTasks IPC backend. rMenu does not open RTasks Quick Add for embedded task capture.

### Panel

User-facing alias:

```text
tasks -> RTasks panel / task list
```

Daemon hotkey:

```text
Ctrl+Space
```

`Ctrl+Space` toggles the RTasks panel when rMenu and the RTasks companion are installed. Closing the panel restores focus to the previously focused window.

The native provider can also show RTasks task rows from:

```text
%APPDATA%\rtasks\tasks.jsonl
```

## rMods registry

`/rmods` is a core-owned command. It is not implemented as a privileged module.

Registry repository:

```text
https://github.com/SynrgStudio/rmods.git
```

Local development checkout:

```text
C:\rmods
```

Default raw registry:

```text
https://raw.githubusercontent.com/SynrgStudio/rmods/main/registry.json
```

Local rMenu files:

```text
<data_dir>\state\rmods-registry-cache.json
<data_dir>\state\rmods-installed.json
<data_dir>\state\downloads\
```

### Module formats

`rmod` is a single UTF-8 module file:

```text
<data_dir>\modules\<id>.rmod
```

`rpack` is a folder module distributed as a file list, not as a zip/archive:

```text
<data_dir>\modules\<id>\
  module.toml
  module.js
  config.json
  README.md
  ...
```

The registry generator scans:

```text
C:\rmods\modules\*.rmod
C:\rmods\rpacks\*
```

`registry.json` is generated by GitHub Actions and should not be edited manually.

### `/rmods` controls

```text
/rmods          open registry list
/rmods <query>  filter registry list
Up/Down         move cursor
Space           mark/unmark pending change
F5/Ctrl+R       refresh registry
Ctrl+U          mark update-available modules
Enter           apply pending installs/updates/removals
Esc             close rMenu
```

Checkbox states:

```text
[x] installed
[ ] not installed
[/] pending change
```

Install security:

- validates module IDs;
- validates paths;
- rejects absolute paths and traversal;
- downloads into state/downloads;
- verifies size and SHA-256;
- installs via staging and atomic move;
- records installed kind/version/checksum in `rmods-installed.json`.

## Module state directories

External JavaScript modules can ask the host for stable paths:

```js
ctx.moduleSourcePath()
ctx.moduleDir()
ctx.moduleStateDir()
```

Use `ctx.moduleStateDir()` for user data. This maps to:

```text
<data_dir>\state\modules\<module-name>\
```

Do not store user data inside an installed `rpack` folder unless the data is disposable, because updates replace package contents.

The `shortcuts` rpack uses `ctx.moduleStateDir()` for:

```text
shortcuts.user.json
shortcuts.pending.json
```

## Color picker rpack

`color-picker` is distributed as an isolated `rpack`, not as rMenu core functionality.

Commands:

```text
color
cp
picker
pick
color hex
color rgb
color hsl
color all
```

It launches the rpack-local helper:

```text
<data_dir>\modules\color-picker\bin\color-picker.exe
```

Behavior:

- native no-console helper;
- floating preview window;
- magnified 15x15 pixel area;
- red center crosshair;
- color chip and text preview;
- click copies the selected color without passing the click to the underlying app;
- Escape cancels;
- holding Shift enables precision mode at 8% mouse speed;
- window avoids active-monitor edges.
